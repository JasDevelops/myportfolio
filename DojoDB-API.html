DojoDB API

A RESTful API for Managing Movies and Users

DojoDB API is a backend service built with Node.js, Express, and MongoDB, designed to handle user authentication and movie data for the DojoDB application. The API enables users to browse movies, register an account, log in, and manage their favorite films.

Project Objective

The goal of DojoDB API is to provide a structured and scalable backend that supports:

  • CRUD operations for managing movies, users, and favorites
  • Secure user authentication and authorization with JWT and Passport.js
  • Efficient database management using MongoDB and Mongoose
  • RESTful endpoints that integrate seamlessly with the DojoDB frontend
  • Cross-origin support for external clients via CORS

Technology Behind DojoDB API

  • Node.js & Express - Backend framework for handling requests
  • MongoDB & Mongoose - NoSQL database for storing movie/user data
  • JWT & Passport.js - Secure authentication and user sessions
  • CORS - Enables cross-origin API access
  • MongoDB Atlas - Cloud database hosting
  • Heroku - Deployment and hosting of the API

Challenges & Growth

Developing the DojoDB API marked my first significant venture into backend development. Previously, my experience was primarily with front-end technologies, so building a RESTful API from scratch was both challenging and enlightening. I gained a deeper understanding of server-side operations, database management, and the intricacies of user authentication. This project definitely expanded my technical skill for handling backend tasks.
