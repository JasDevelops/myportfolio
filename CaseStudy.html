Case Study – DojoDB

Introduction

Project Overview

DojoDB is a movie database web application developed as part of the CareerFoundry Full-Stack Web Development program. It features:

  • A RESTful API (using Node.js/Express and MongoDB) for managing movie data and user authentication.
  • A React-based frontend that lets users browse, filter, and save their favorite movies.

Purpose & Context

The primary goal was to master full-stack development by building a complete application from scratch within a guided framework. The project was divided into two phases:

  • Backend (Nov 15 – Dec 6): Developed a secure API with user registration and data management.
  • Frontend (Dec 9 – Jan 11): Created a responsive React interface with movie filtering and a favorites feature, separated by a brief holiday break.

Objective & Role

The objective was to build a robust full-stack solution that demonstrates technical proficiency and clear information presentation. I developed DojoDB as the sole developer under CareerFoundry’s guidance over roughly two months.

Process

Planning & Setup

  • Outlined the overall architecture and defined key functionalities by mapping API endpoints and data models.
  • Created wireframes and flowcharts to visualize the user journey—especially for filtering and favoriting features.

Backend Development

Constructed a secure RESTful API using Node.js/Express and MongoDB, supporting user registration, login, and data management.

  • User Authentication: Implemented token-based authentication for secure sessions.
  • Data Structuring: Designed scalable data models for movies and users.

Frontend Development

Built a dynamic, responsive React interface that allows users to browse, filter, and save their favorite movies.

  • State Management: Utilized React hooks to manage dynamic content and user interactions.
  • Responsive Design: Ensured an optimized UI across desktops and mobile devices.

Testing & Iteration

  • Conducted regular manual testing and debugging to resolve issues in both API endpoints and React components.
  • Made iterative improvements by optimizing API responses, streamlining UI interactions, and refining the favorites feature.

Conclusion

Final Outcome & Results

DojoDB culminated in a fully functional movie database that seamlessly integrates a secure RESTful API with a responsive React frontend. The application enables users to browse, filter, and save their favorite movies.

Challenges & Lessons Learned

  • User Authentication & Data Structuring: Implementing secure, token-based authentication and designing scalable data models deepened my backend expertise.
  • State Management & Responsive Design: Ensuring smooth interactions and optimal display across devices highlighted the need for iterative testing and refinement.

Future Directions

  • Database Expansion: Add a broader range of movie entries to offer users a more diverse selection.
  • Enhanced Engagement: Introduce features such as user reviews or recommendations.
  • Performance Optimization: Further refine API responses and frontend performance for increased scalability.
